# 织境故事

> 近未来，你和你的 Agent 会怎么协作？

## 定位

这些故事不是科幻。它们是把框架里的理论放进"如果明天就这样了"的场景里，让你**感受到**而不是**理解到**。

每个故事基于真实的小组织AI需求调研，用具体的角色和情境展示框架模块在现实中意味着什么。

## 写作规范

### 元数据格式

每篇故事开头必须包含：

```markdown
---
title: 故事标题
architecture:
  - 02-economy          ← 对应 framework/ 下的模块编号
  - 11-lifecycle
status: draft | published
author: 作者名
---
```

### 写作原则

1. **近未来，不远未来** — 场景设定在1-2年内可能发生的现实
2. **需求先行** — 每篇故事基于真实的小组织AI痛点调研
3. **人比概念重要** — 读者记住的是角色和情境，不是术语
4. **短** — 一个场景讲透一件事
5. **可以矛盾** — 不同故事对同一模块的理解可以不同
6. **场景多样** — 不同故事使用不同类型的小组织作为背景
7. **结尾克制** — 不升华、不鸡汤，留白让读者自己想

### 不需要

- 不需要连续的剧情线
- 不需要统一的时间线
- 不需要角色一致性
- 不需要审批（PR 即可）

## 贡献新故事

1. 选择一个你想展示的框架模块
2. 找到真实的小组织AI需求作为背景
3. 选择一个组织类型（培训机构、独立咖啡店、自由职业联盟等）
4. 写一个场景，让模块的核心论点自然呈现
5. PR 提交，标题格式：`story: 故事标题`

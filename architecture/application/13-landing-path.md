# 13 落地路径 (Landing Path)

## 核心问题

**一个关于 AI 原生组织的思想实验，如何从理论走向可验证的现实？**

---

## 前提假设

1. 宏大叙事的落地方式不是"一步到位"，而是"单点打透→横向复制→纵向整合"。
2. 最小可行验证点（MVP）必须满足：可观测、可度量、可迁移。
3. "农村包围城市"——从边缘场景切入，避开与主流组织范式的正面竞争。
4. 每个阶段都有明确的退出条件和失败标准。

---

## 核心论点

### 论点一：三阶段落地模型

```
阶段一：顾问模式（单点打透）
  ↓ 验证条件：MVP组织的AI化运营指标显著优于传统运营
阶段二：同类型复制（圈子扩散）
  ↓ 验证条件：3个以上同类型组织独立运行，方法论可复制
阶段三：平台化（生态成型）
  ↓ 验证条件：跨类型组织自发形成，治理和声誉系统可跨组织运转
```

### 论点二：阶段一——顾问模式（现在 → 6个月）

**目标**：用一个真实案例证明"顾问+AI工具"能为小型组织带来10倍效率提升。

**载体**：摇摆舞组织。

**为什么是摇摆舞？**
- 组织足够小（核心团队3-5人），变量可控。
- 运营痛点清晰（报名管理、内容传播、新人转化）。
- 社区属性强，天然适合"声誉"和"贡献"的量化。
- 顾问（项目发起人）与组织之间有深度信任关系，降低了"被当作实验品"的阻力。

**具体交付物**：
1. 摇摆舞组织的 AI 辅助运营系统（报名自动化 + 内容分发 + 新人引导）。
2. 顾问介入前后关键指标的对比报告。
3. 方法论文档（FDE模型在真实场景中的完整走查）。

**退出条件**：
- 系统运行3个月，组织核心指标（报名转化率、社区活跃度）提升50%以上。
- 顾问可以撤出，组织内部人员能独立维护系统。

**失败标准**：
- 组织拒绝采用系统，或采用后3个月内指标无显著改善。
- 这不意味着整个项目失败，但意味着 MVP 载体选择有误，需要换场景重新验证。

### 论点三：阶段二——同类型复制（6个月 → 18个月）

**目标**：将阶段一的方法论复制到3-5个同类型的小型社区组织（其他舞蹈社群、独立音乐人圈子、手工艺人协会等）。

**为什么是"同类型"而不是"跨类型"？**
- 同类型组织的痛点高度相似，方法论迁移成本低。
- 可以快速积累同一领域的"模式库"，提升顾问效率。
- 跨类型复制是阶段三的事——过早跨界会分散精力。

**关键动作**：
1. 从阶段一的顾问经验中提取"可复制的顾问流程"（标准化 FDE 的执行手册）。
2. 培训2-3名新顾问（从探索者中选拔）。
3. 为每个新组织建立独立的运营看板和指标体系。
4. 开始积累跨组织的"知识库"雏形。

**退出条件**：
- 3个以上组织独立运行且指标正向。
- 至少有1名新顾问可以独立完成从诊断到方案交付的全流程。
- "可复制顾问流程"文档经过至少3次独立验证。

**失败标准**：
- 复制成功率低于50%（超过一半的新组织无法跑通流程）。
- 新顾问培训后仍无法独立交付，说明方法论过于依赖个人能力。

### 论点四：阶段三——平台化（18个月 → 36个月）

**目标**：从"顾问驱动"转向"平台驱动"。织境本身成为组织创建、运营、治理的基础设施。

**关键转变**：
- 顾问从"核心服务提供者"变为"培训师和标准化者"。
- 组织可以自助完成创建、配置AI工具、招募成员。
- 跨组织的声誉系统开始运转。
- 治理机制（调解、仲裁）开始被需要——因为有足够多的组织和互动。

**平台的核心功能**：
1. **组织创建引擎**：模板化的组织配置（选择行业、规模、治理模型），一键生成初始AI工具链。
2. **知识市场**：锻造师产出的知识模块可以跨组织交易。
3. **声誉网络**：个人的Rep跨组织可视化和累积。
4. **治理工具箱**：调解流程的数字化、仲裁记录的透明化。

**这一阶段的风险**：
- 平台化可能过早——如果阶段二的验证不充分，平台会建立在沙上。
- 治理机制的引入可能引发"制度恐慌"——小型组织不习惯被治理。
- 跨组织的声誉系统面临"度量衡"问题——不同组织的Rep是否等价？

## 理论论证

落地路径的设计遵循**精益创业（Lean Startup）**的核心逻辑，但有一个关键修正：织境的 MVP 不是产品，而是**方法论**。

传统精益创业验证的是"产品-市场匹配"（PMF）。织境验证的是"方法论-场景匹配"（MSF）：FDE顾问方法论能否在特定场景中产生可量化的价值。

"农村包围城市"的逻辑来自**创新扩散理论（Everett Rogers）**。Rogers 将采纳者分为创新者、早期采纳者、早期多数、晚期多数、落后者。织境的阶段一瞄准"创新者"（摇摆舞组织），阶段二瞄准"早期采纳者"（同类型社区），阶段三才能触及"早期多数"。

跳过阶段一直接平台化，等同于瞄准"早期多数"——而他们不会为一个未经验证的系统买单。

## 开放问题

1. 阶段一的 MVP 如果失败，是否应该换场景还是换方法论？判断标准是什么？
2. 阶段二"同类型"的边界在哪？舞蹈和音乐算"同类型"吗？舞蹈和手工艺呢？
3. 平台化后，织境本身会不会变成一个需要被治理的"超级组织"？
4. 如果阶段二的复制速度极快（3个月内覆盖10+组织），是否可以跳过部分验证直接进入阶段三？

---

## 可证伪条件

1. **阶段一失败**：摇摆舞 MVP 在3个月内无法产生可量化的效率提升，且原因不是执行问题而是"这个场景根本不需要 AI"。
2. **方法论不可复制**：阶段二中，新顾问即使经过培训也无法复现阶段一的结果，说明"顾问能力"是个人天赋而非可传授的方法论。
3. **平台化无需求**：阶段三启动后，没有组织愿意自助创建，仍然依赖顾问介入，说明"自助平台"的前提假设不成立。

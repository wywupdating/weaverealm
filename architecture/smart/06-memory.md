# 06 — 记忆系统

## 核心问题

AI 组织如何记住自己的经验——是靠一个无限大的上下文窗口，还是靠一套会遗忘、会整理、会做梦的记忆架构？

## 前提假设

1. **记忆≠存储**。把信息存下来不等于记住了。记忆的本质是"在需要时能被有效调用"，这要求信息经过组织、压缩、索引，而非原始堆积。
2. **无限上下文也不能替代分层**。即便未来上下文窗口扩展到千万 token，三个结构性问题依然存在：O(n²) 的注意力计算复杂度、"Lost in the Middle" 的信息衰减、以及信噪比随上下文长度退化。分层不是权宜之计，而是架构必然。
3. **组织的记忆应该像大脑，不应该像硬盘**。硬盘是"写入→存储→读取"的线性模型。大脑是"编码→巩固→再巩固"的动态模型，记忆会在每次回忆时被重新写入和修正。

## 核心论点

### 论点一：三层架构——L1 索引层 / L2 展开层 / L3 执行层

```
L1 索引层（索引卡片）
  └─ 存储什么：经验的摘要、标签、关联关系
  └─ 特征：高压缩比、常驻内存、O(1) 检索
  └─ 类比：海马体——快速编码，不存细节

L2 展开层（完整记录）
  └─ 存储什么：经验的完整上下文——当时发生了什么、做了什么决策、结果如何
  └─ 特征：按需加载、支持上下文重建
  └─ 类比：新皮层——长期存储，需要时调取

L3 执行层（工作记忆）
  └─ 存储什么：当前任务所需的活跃信息
  └─ 特征：容量有限、高频更新、任务结束即清理
  └─ 类比：前额叶皮层——当前注意力的焦点
```

三层之间的关系不是"存储层级"（如 L1/L2/L3 Cache），而是"认知层级"：L1 回答"我有没有相关经验"，L2 回答"那次经验的具体细节是什么"，L3 回答"现在该怎么做"。

### 论点二：Thread-Agent 模型——线程是数据层，Agent 是计算层

```
Thread（线程）= 一段连贯的经验数据
  └─ 有明确的开始和结束
  └─ 包含输入、决策过程、输出、反馈
  └─ 可被独立索引和检索

Agent = 一组线程的执行者和继承者
  └─ 创建时绑定到线程
  └─ 执行中写入线程
  └─ 合并时将线程沉淀到 L2，索引沉淀到 L1
```

关键洞察：**将数据和计算解耦**。线程可以被不同的 Agent 读取（知识共享），Agent 可以在不同线程间迁移（经验复用）。如果 Agent 就是数据本身，这两个操作都不可能。

### 论点三：事件驱动再巩固

人脑的记忆不是一次写入就定型的。每次回忆都会触发"再巩固"（reconsolidation）——记忆被提取、可能被修改、然后重新存储。这个过程不是缺陷，而是特性：它让记忆能够被新经验持续更新。

织境的记忆系统采用同样的机制：

- **触发条件**：当新经验与已有记忆的 L1 索引高度匹配时（相似度超过阈值），触发再巩固。
- **执行过程**：提取 L2 中的旧经验，与新经验合并，重新生成摘要和索引，写回 L1 和 L2。
- **结果**：记忆不是越来越长，而是越来越准确。噪声被滤除，模式被强化。

### 论点四：为什么无限上下文不够

三个结构性论据：

1. **计算复杂度**。Transformer 的标准注意力机制是 O(n²)。上下文从 8K 扩展到 1M，计算量增长 16000 倍。即使硬件跟上，这个增长也是不可持续的。
2. **Lost in the Middle**（Liu et al., 2023）。实验证据表明，LLM 对上下文中间部分的信息检索准确率显著低于开头和结尾。上下文越长，中间区域的"信息盲区"越大。
3. **信噪比退化**。在 1M token 的上下文中，与当前任务相关的信息可能只有 1K token。信噪比是 1:1000。分层记忆系统的目标是把信噪比提升到 1:10 甚至更高。

## 理论论证

**神经科学映射**。三层架构直接对应记忆神经科学中的标准模型（Standard Model of Memory Consolidation, McGaugh 2000）：海马体快速编码（L1）→ 新皮层长期存储（L2）→ 前额叶工作记忆（L3）。这不是隐喻，而是功能同构。

**计算复杂度理论**。分层索引是信息检索的经典解法。从 B-Tree 到 LSM-Tree，核心思想都是"用层级换性能"。把同样的思想应用到 LLM 上下文管理，逻辑上没有跳跃。

**Codex 架构验证**。Codex 的上下文管理本质上是一个两层的记忆系统：当前文件上下文（L3）+ 项目索引（L1/L2 混合）。它不尝试把整个代码库塞进上下文，而是用索引定位、按需加载。这证明了分层方案在工程上是可行的。

## 与其他模块的关系

- **被 05-Agent架构依赖**：Agent 的生命周期管理（合并、唤醒）直接调用记忆系统的读写接口。
- **依赖 07-验证门控**：写入 L2 的经验应该是经过验证的，未通过验证的经验进入 rejected-edit buffer 而非主记忆。
- **被 08-空间智能化依赖**：每个空间的"空间智能"本质上是该空间的记忆系统实例。

## 开放问题

1. **失败经验的记忆维度**。当前三层架构天然倾向记录"成功的经验"（因为成功会产生可索引的输出），但失败经验（什么不该做）同样重要。这个维度目前在架构中是缺失的。
2. **L1 索引的漂移问题**。随着再巩固的反复发生，L1 索引是否会逐渐偏离原始经验？需要一个"保真度校验"机制。
3. **跨空间记忆共享的隐私边界**。Agent A 在空间 X 中的经验，多大程度上可以被 Agent B 在空间 Y 中引用？全量共享会导致信息泄漏，完全不共享会丧失组织学习能力。模块 05 的"版本化同步仓库"提供了一种中间方案：通过权限控制（只读/可修改/协商修改）在共享和隐私之间取平衡，但最优的默认权限策略尚未确定。

## 可证伪条件

1. 如果未来的 LLM 架构（如 Mamba、RWKV 等线性注意力模型）成功解决了 O(n²) 和 Lost in the Middle 问题，且信噪比退化被工程手段（如 sparse attention）有效控制，那么分层的必要性将被大幅削弱。
2. 如果事件驱动再巩固在实践中被证明"改写"了关键事实（记忆准确性下降而非上升），那么再巩固机制就需要加入不可变锚点（immutable anchors），架构复杂度会显著增加。
3. 如果 Thread-Agent 模型的解耦在实践中导致 Agent 无法维持连贯的"组织身份感"（组织成员感觉 AI "不认识自己了"），那么数据和计算可能需要部分耦合。

# 个人助理发展方向思考（2026-06-08）

## 核心洞察
- 企业Agent（Codex/WorkBuddy）和B2C个人Agent是两条并行赛道
- 随着记忆系统突破，个人助理型Agent已接近可用
- 个人助理→组织责任Agent是同一路径的不同规模实例化

## 架构模型：主身+分身+装备
- **主身（白衣）**：核心身份，持有完整记忆和信任关系，唯一和用户对话的入口
- **分身**：主身在不同场景的实例，穿不同装备，同时执行不同任务
- **装备**：场景相关的技能包、知识库、权限边界、行为模式
- 用户只维护一个关系（和主身），所有复杂度内部消化

## 待解决的关键问题
1. **知情阈值**：主身不必过每条消息，但需要设计什么级别必须过、什么级别分身可自主
2. **跨分身冲突裁决**：工作分身vs生活分身的资源冲突，主身需要跨上下文判断
3. **零配置场景识别**：普通人不会手动定义空间边界，Agent需要自动判断场景
4. **调度哲学**：等价于AI工会理论的Rep/Share/DKP，个人助理体系也需要自己的经济/调度模型

## 决策
- 不在大方向上继续展开，先把手头事情做好
- 织境项目是这些想法的真实实验场——主身调度、记忆分层、项目间协调
- 做完自然有答案

## 关联
- 与AI工会理论（游戏×AI）的关系待厘清
- 与Agent World身份注册的思路一致：同一身份在不同场景扮演不同角色

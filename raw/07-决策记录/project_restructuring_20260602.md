# 项目拆分与合并决策（2026-06-02）

## 决策背景
用户原本有多个项目文件夹，但都是Agent本地文件，与扣子3.0的项目空间是两套体系。为了更好地利用扣子3.0的项目空间功能，需要对现有项目进行拆分与合并。

## 核心决策
### 项目拆分与合并
1. **保留并升级为扣子项目空间的项目**：
   - 游戏×AI：最活跃、内容最重、理论体系最完整，配置理论碰撞Agent、研究Agent
   - 职业发展：将简历优化助手、职业发展、个人能力模型合并，配置求职+简历优化Agent
   - Claude Code学习：日程驱动持续进行，配置出题Agent、学习助手

2. **砍掉的项目**：
   - 鸿蒙智能体：本地文件留作参考，不建扣子项目空间

3. **留本地不建项目的**：
   - 健身与作息：日程驱动，不需要项目空间
   - 服务器运维：工具性参考
   - 办公矩阵：角色定义，系统级
   - 学习资料：散装，随时可搜
   - 核心循环、日记、热点追踪：系统级，不属于任何项目

## 合并逻辑
简历优化本质是职业发展的一个子任务——找工作=简历+岗位匹配+能力分析，放一个项目里Agent可以串起来用。

## 执行步骤
1. 逐个创建扣子项目空间
2. 把对应本地文件迁移到项目文件系统
3. 为每个项目配置专属Agent
4. 更新MEMORY.md和日程，让后续日程任务指向新项目路径

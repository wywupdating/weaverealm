# 契约驱动分层Agent架构（2026-05-29 感悟）

## 背景

大型企业使用Agent的核心矛盾：效率是第二优先级，准确性才是第一优先级。当前Agent的幻觉和模糊输出在生产环境中是灾难。

## 架构设计

三层架构：
1. **Worker Agent**：固定输入→固定输出，不允许自主调整结果。可以看到更多上下文（Read-Only Context），但只能输出log/警告，等待上级或人工校准
2. **工长Agent**：负责一条流水线上产出结果的核对和检查，核心是比对（校验）不是推理（重新生成）
3. **全局Agent**：对工长Agent负责，做跨流水线协调和最终决策

## 关键原则

- 输出契约比输入契约更重要
- 上下文只读，权限隔离：判断质量不降，副作用可控
- 分层本质是缩小每层决策空间
- 不让Worker自作主张，只给log不给action
- 金融、医疗、法律等容错率极低的场景才是正确设计

## 与现有架构对比

- Claude Code sub-agent是任务委派型（Worker有自主决策权）
- 本方案是契约约束型（Worker无自主决策权，只有固定IO和log）
- OpenAI Symphony、Claude Code+Codex Lead-Worker都在往这个方向靠
- 本方案约束更严格但更适合企业生产环境

## 一句话总结

Agent的聪明应该体现在发现问题时及时警告，而不是自作主张去修正。

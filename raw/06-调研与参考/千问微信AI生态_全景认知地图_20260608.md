## 一、千问Agent生态全景

### 1.1 定位与战略

阿里正从「基座模型追赶者」向「Agent生态构建者」跃迁。AI竞争已从参数比拼升级为 **"底层模型 + 中层Agent框架 + 上层入口矩阵"** 的综合体系作战 [^1]。

- **核心定位**：千问云 = 为Agent而生的AI模型服务平台
- **战略优先级**：阿里成立Token Foundry事业部，CEO直管
- **入口矩阵**：千问APP + 通义灵码 + Devs + CodeWork + 千问云API
- **差异化打法**：凭借合规性红利 + 本土服务生态，与海外Anthropic等错位竞争

### 1.2 技术栈分层

| 层次 | 组件 | 说明 |
|------|------|------|
| **模型层** | Qwen3.7-Max（旗舰）/ Qwen3.7-Plus（多模态Agent）/ Qwen3-14B（开源本地） | 150+主流模型API，含GLM/Kimi/DeepSeek等 [^2] |
| **Agent框架层** | AgentScope Java 2.0 / qwen-agent(Python) | 企业级分布式部署，Spring Boot集成 |
| **工具/技能层** | 千问云 Skills + CLI | 模型选型/调用/认证/用量查询全链路封装 |
| **渠道层** | 千问APP/千问云API/第三方Agent框架 | 兼容OpenAI协议、Claude Code、OpenClaw、Qwen Code |
| **开放能力** | 第三方Agent/Skills全面开放 | 企业可在千问APP内运营自有品牌Agent [^3] |

### 1.3 开发者接入路径

**路径A：模型API调用（最轻量）**
- 阿里云百炼平台 → API Key → OpenAI兼容协议调用
- 端点：`https://dashscope.aliyuncs.com/compatible-mode/v1`
- 定价：Qwen3.7-Plus 输入$0.4/百万token，输出$1.6/百万token [^4]

**路径B：千问云Skills/CLI（Agent原生）**
- Skills：模型选型/调用/认证/用量全链路封装，Agent无需写代码即可调用
- CLI：登录/查询/调用/用量/诊断，脚本或命令行自动化
- 均已GitHub开源

**路径C：AgentScope Java（企业级）**
- HarnessAgent框架：workspace-driven自进化
- 写一套Agent逻辑 → 本地单机(Claw) ↔ 企业分布式(Builder) 自由切换
- 核心概念：AGENTS.md + skills/ + subagents/ + MEMORY.md 双层记忆
- 6个开箱即用渠道（钉钉/企微/飞书/Web/API/微信）[^5]

**路径D：品牌Agent入驻（面向企业）**
- 6月3日起全面开放第三方Agent/Skills [^3]
- 首批：瑞幸/肯德基/蜜雪冰城/东航
- 企业定制Agent人设、服务能力、边界，在千问APP内运营
- 本质：AI时代的新运营阵地（继App/公众号/小程序之后）

### 1.4 能力边界与限制

- **模型差距**：国内头部模型间差距收敛至约10分，但与海外Anthropic在编程领域差距拉大
- **多模态Agent**：Qwen3.7-Plus屏幕理解79分（超GPT-5.4的67.4），但复杂页面仍有乱码/交互失效/黑屏问题 [^6]
- **Agent框架**：AgentScope Java面向企业Java团队，对非Java开发者有门槛
- **生态成熟度**：品牌Agent刚起步，首批仅消费生活服务类

### 1.5 机会窗口判断

| 时机 | 信号 | 行动 |
|------|------|------|
| **现在** | 千问云Skills/CLI已开源，API已开放 | 学习API调用+Skills开发，跑通demo |
| **6-12月** | 品牌Agent生态扩展至更多垂直行业 | 评估是否为组织开发千问Agent |
| **1-2年** | Agent交易市场/付费能力完善 | 如果做工具平台，这是入驻时机 |

## 二、微信AI生态全景

### 2.1 定位与战略

微信正进化为「超级AI生态入口」。14亿月活 + 数百万小程序 = AI调用所有服务的终极形态 [^7]。

- **核心定位**：AI智能体通过自然语言调用小程序完成订餐/预约/下单等全场景
- **战略优先级**：腾讯最高战略优先级
- **入口设计**：微信主界面右滑调出AI助手对话窗口
- **关键动态**：6月8日正式发布《开发者接入微信AI生态指引》[^8]

### 2.2 技术栈分层

| 层次 | 组件 | 说明 |
|------|------|------|
| **模型层** | 腾讯混元2.0（主力）/ 智谱/DeepSeek等（已测试） | 混元在行业测评与头部有差距 [^9] |
| **AI能力层** | 微信AI智能体 | 通过页面交互技术代替用户执行搜索/下单/输入等操作 |
| **小程序层** | 数百万小程序 | AI通过"技能"调用标准化接口操作 |
| **接入层** | 自动模式 + 开发模式 | 两种不互斥，可同时开启 |
| **渠道层** | 微信APP内（右滑唤起）/ 手机厂商AI助手（A2A协议） | 荣耀已接入，华为/小米/OPPO/vivo跟进中 |

### 2.3 开发者接入路径

**路径A：自动模式（零代码）**
- 小程序管理后台 → AI能力 → 授权自动模式
- 平台提审时读取源码 → 自动分析页面 → 微信AI直接操作
- 适合：已有小程序、想快速被AI调用的团队

**路径B：开发模式（个性化）**
- 基于小程序业务特性自主开发
- 通过平台评测与审核 → 可被微信AI调用
- 适合：需要精细控制AI交互逻辑的团队

**路径C：AI小程序成长计划（从零开始）**
- 激励：1亿Token混元2.0额度 + 6个月免费云开发 + 流量激励
- 接入：云开发平台直接调用混元模型 或 服务端直连
- 支持：安卓/iOS/鸿蒙全终端虚拟支付+会员订阅
- 入口：小程序后台 → 行业能力 → AI小程序成长计划 [^10]

**路径D：iLink Bot API（个人微信Bot）**
- 腾讯官方通过OpenClaw框架开放微信个人账号Bot API
- 域名：ilinkai.weixin.qq.com
- 社区方案：cc-weixin（约200行核心代码桥接微信+Claude Code Agent）
- 注意：个人终端部署有数据安全风险 [^11]

### 2.4 能力边界与限制

- **模型能力**：混元与头部模型存在差距，微信团队已测试其他厂商模型
- **语音识别**：小程序无流式ASR，端到端延迟1-3秒，实时交互场景受限
- **生态整合**：小程序接口/数据格式/业务逻辑碎片化，跨小程序任务编排难度极高
- **内测阶段**：功能尚未对用户开放，开发者先接入、用户排队等
- **封闭生态**：免费Token绑定混元模型，本质是"补贴换规模"

### 2.5 机会窗口判断

| 时机 | 信号 | 行动 |
|------|------|------|
| **现在** | 开发者指引已发布，内测阶段 | 如有小程序 → 立即接入自动模式占位 |
| **3-6月** | 灰度测试启动，首批用户体验 | 关注用户反馈，优化AI交互体验 |
| **6-12月** | AI小程序竞争白热化 | 如果做AI工具 → 这是微信生态的早期红利窗口 |
| **1-2年** | 微信AI成为标准入口 | 如果有垂直领域积累 → 做该领域的微信AI工具服务商 |

## 三、三大Agent平台横向对比

| 维度 | 千问Agent生态 | 微信AI生态 | 扣子Coze（你在用的） |
|------|-------------|-----------|-------------------|
| **背后公司** | 阿里巴巴 | 腾讯 | 字节跳动 |
| **核心壁垒** | 模型能力+企业Agent框架 | 14亿用户+数百万小程序 | 低代码+多端协同+技能商店 |
| **开发者门槛** | 中等（API→Skills→AgentScope分级） | 低（自动模式零代码） | 极低（拖拽式+对话创建） |
| **模型开放性** | 150+模型API，兼容OpenAI协议 | 混元为主，正测试其他 | 豆包为主，第三方接入有限 |
| **Agent框架** | AgentScope Java 2.0（企业级） | 无独立框架，依赖小程序生态 | 内置工作流编排 |
| **多Agent协作** | AgentScope支持分布式多Agent | 单AI调度多小程序 | 项目空间内多Agent协作 |
| **分发渠道** | 千问APP/阿里生态 | 微信APP（14亿用户） | 豆包/飞书/微信/网页/API |
| **私有化部署** | 支持（AgentScope Builder） | 不支持 | 不支持 |
| **商业模式** | API按量+Token Plan订阅 | 补贴期（免费Token+云开发） | 免费基础+订阅会员 |
| **成熟度** | 中（品牌Agent刚起步） | 早期（内测） | 相对成熟（3.0版本） |
| **你的机会** | 技能开发/品牌Agent服务商 | 小程序AI化/垂直工具 | 已在用，持续深耕 |

## 四、对你的战略判断

### 4.1 你当前的定位

你在扣子(Coze)上已经搭建了完整的工作流（Max+子Agent+日程+记忆+项目空间），这是你的主场。千问和微信AI目前对你来说是**认知储备**，不是迁移目标。

### 4.2 三个反直觉判断

**判断1：千问生态的核心价值不是模型，是Agent基础设施**
- AgentScope Java + 千问云Skills/CLI 构成了一套从个人到企业的Agent全栈方案
- 如果你未来要给组织做AI工具（摇摆舞MVP等），千问的Agent框架比Coze更适合企业部署
- 但现在不急，等MVP验证后再看

**判断2：微信AI的机会不在"做小程序"，在"帮别人做小程序AI化"**
- 数百万小程序运营者不会自己接AI → 这是服务商机会
- 你的游戏×AI方向（帮现有组织做AI工具）天然对齐这个需求
- 自动模式零代码接入 → 你可以帮摇摆舞组织快速接入微信AI

**判断3：三大平台不互斥，是组合拳**
- Coze = 你的个人生产力工具（已在用）
- 千问 = 企业级Agent部署的备选技术栈
- 微信AI = 面向C端用户的分发渠道
- 最优策略：用Coze做原型验证 → 用千问AgentScope做企业部署 → 用微信AI做C端触达

### 4.3 近期学习建议

1. **千问**：注册阿里云百炼，跑通API调用 → 理解Skills/CLI机制 → 读AgentScope Java文档
2. **微信AI**：如有小程序 → 接入自动模式占位 → 关注iLink Bot API动态
3. **两个追踪项目已启动**：每天20:00自动推送最新动态，保持认知更新

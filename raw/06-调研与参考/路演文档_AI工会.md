## 一、项目概述（Executive Summary）

### 一句话定义

**AI工会**是一种以知识/技能沉淀为核心、以AI为组织运转地基的新型协作组织形态——人进来贡献和获取，AI承载整个组织的记忆、规则和运作。

> *"未来的AI公司，它其实并不在意在这个AI公司里工作的具体是哪些人。他更在意的是他有没有吸收到足够独属于这个公司本身的知识和技能。当这个AI公司拥有足够的针对它业务的知识和技能之后，AI其实本身已经可以自运转了。"*
> —— 大白（王宇为）

### 核心创新点

| # | 创新点 | 说明 |
|---|--------|------|
| 1 | **AI做地基，不做工具** | 所有现有产品（Viktor、aweb.ai、元宝派）把AI放在"助手"或"替代者"的位置。AI工会的AI是组织运转的基础设施，人在AI之上贡献和获取 |
| 2 | **知识是核心资产，人不是** | 人员可进出，但贡献的知识和技能永久沉淀在工会。这一点的实现方式是DKP贡献系统——贡献量化→权益对应→可变现 |
| 3 | **分配机制重构** | DKP经济系统实现透明化分配：创建者永久10%上限、分红+退出双重变现、贡献可超越先来者——可能是比公司更公平的组织形态 |

## 二、问题与痛点

### 当前公司形态的三大困境

#### 2.1 知识库困境：输入无动力，输出质量差

**输入端**：员工不愿意贡献知识。

原因很现实——"我把所有经验都写进公司知识库，公司转头把我开了怎么办？"在传统公司里，知识=权力，贡献知识=削弱自己的不可替代性。华为为什么能在知识管理上做得好？因为任正非愿意把"知本"变成"资本"，让知识贡献者有回报。但这是反人性的例外，不是常态。

**输出端**：即使有人愿意贡献，公司知识库的数据质量依然很差。

- 文档散落在个人电脑里（"老王知道怎么搞定这个客户，但老王走了知识就带走了"）
- 碎片化数据对AI不友好（RAG效果差）
- 结构化整理的成本极高（华为花了20年）

**大白原话**：
> *"微信群里大家自发分享视频、感悟、知识点，但这些东西没法维护，看一眼就消失了。自发贡献的动力在很多小型组织里天然存在，只是之前没法凝聚起来。"*

#### 2.2 人才绑定困境：人走了什么都没留下

公司最大的资产是"人"，但人是最不可控的资产。

- 员工离职 = 带走隐性知识（判断力、经验、人脉）
- 核心员工创业 = 带走客户和团队
- 公司为"人才"支付高薪，但买不到知识的所有权

**知识黑洞**：
1. 经验私有化：只有老王知道怎么搞定这个客户
2. 流程隐形化：口头流程，新人反复犯错
3. 复盘形式化：开完会写个总结就结束

#### 2.3 分配不公困境：资本决定一切，后进者天花板明显

公司形态天然导致财富向头部集中——九死一生把公司搭起来的人，未来吃一辈子赚几千亿上万亿。

- 股权激励是少数人的游戏
- 员工拿固定工资，承担公司风险，但不分享公司增长
- 后进者无论贡献多大，追不上创始人的股权差距

### 目标用户的具体场景

**场景1：舞蹈社群组织者（创始人的亲身经历）**

- 微信群200人，每周有新人问"这个动作怎么做"
- 老人愿意发视频教学，但发了就沉了
- 组织者想把知识沉淀下来，但：
  - 没有工具（手动整理费时费力）
  - 没有动力（沉淀了知识对自己有什么好处？）
  - 没有持续性（人走了知识就没了）

**场景2：技术团队负责人**

- 团队10-20人，新人入职需要3个月才能上手
- 老员工的知识在脑子里，没法传承
- AI工具买了不少，但数据质量太差用不起来

**场景3：知识型个体户/超级个体**

- 一个人+AI工具=一家公司（OPC模式爆发）
- 但知识沉淀靠自己，无法复用和变现
- 想招募帮手但雇佣成本高、协作难

## 三、解决方案：AI工会

### 3.1 AI工会的核心定义

**AI工会**是一种以知识/技能沉淀为核心、人员可进出的自生长组织。AI不是工具，也不是替代者，而是组织运转的地基（Foundation）。

```
┌─────────────────────────────────────────────────────────────┐
│                         AI 地基                             │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐       │
│  │   知识沉淀   │  │   规则执行   │  │   记忆传承   │       │
│  └─────────────┘  └─────────────┘  └─────────────┘       │
├─────────────────────────────────────────────────────────────┤
│                      人 在 AI 之 上                          │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐       │
│  │   贡献技能   │  │   获取知识   │  │   积累权益   │       │
│  └─────────────┘  └─────────────┘  └─────────────┘       │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 与传统公司的本质区别

| 维度 | 传统公司 | AI工会 |
|------|---------|--------|
| **核心资产** | 人（员工） | 知识/技能（沉淀在AI） |
| **人员定位** | 人是生产力 | 人是贡献者 |
| **知识归属** | 公司所有，员工带不走 | 留在工会，但贡献者有分成 |
| **退出机制** | 离职手续，可能有竞业限制 | 出售DKP换取分成，带走自己的收益 |
| **分配逻辑** | 固定工资+股权（少数人） | DKP量化+分红+退出变现（所有人） |
| **组织扩张** | 雇佣更多人 | 吸引更多贡献者 |
| **AI角色** | 工具（人用AI） | 地基（AI承载人） |
| **适用规模** | 中大型组织 | 小团体起步，可聚合扩展 |

### 3.3 AI工会的自增强效应

```
贡献者加入 → 提供知识和技能 → AI能力增强 
     ↑                                    ↓
     ←←← 吸引更多贡献者 ←←← 工会更强
```

- **工会越强大** → 吸引越强的人 → 贡献越好的技能 → 工会更强
- **能力越集中** → 不可替代性越强 → 议价能力越高 → 吸引更多客户

### 3.4 产品形态演进路径

| 阶段 | 形态 | 核心功能 | AI角色 |
|------|------|---------|--------|
| **MVP（0-6月）** | 小团体AI工具包 | 知识沉淀、简单DKP计数、AI自动整理 | 助手+工具 |
| **成长（6-18月）** | 多组织平台 | 工会创建、贡献体系、分红系统 | 助手+基础设施 |
| **成熟（18-36月）** | 工会生态联邦 | 跨工会协作、联邦治理、开放API | 核心基础设施 |
| **终态（3年+）** | AI原生组织形态 | 动态治理、权力衰减、反垄断机制 | 治理层+执行层 |

## 四、产品规划（Roadmap）

### Phase 1：轻工具切入（0-6个月）

**目标**：验证最小闭环——组织者愿意付费、成员愿意贡献、AI能沉淀价值

**核心功能**：
1. **工会创建**：5分钟创建工会，上传Logo、名称、简介
2. **知识沉淀**：成员提交视频/文章/链接，AI自动分类和打标签
3. **DKP计数**：记录每个成员的贡献行为（提交、评论、引用）
4. **简单查询**：新人问"这个怎么做"，AI基于知识库回答

**目标用户**：
- 舞蹈社群组织者（创始人的第一个MVP场景）
- 小型培训师/知识创作者
- 10-50人的兴趣小组

**冷启动定价**：
- 原价¥5000/年的组织套餐
- 种子用户优惠价¥2000/年
- 包含：10个成员席位、AI知识库、基础DKP系统

**成功标准**：
- 签约10个种子组织者
- 月活跃成员数>100
- 知识沉淀量>1000条

### Phase 2：组织者生态（6-18个月）

**目标**：飞轮启动——用的人越多，越多人想加入

**核心功能**：
1. **贡献体系升级**：DKP分红（季度）、贡献等级、解锁权限
2. **AI能力升级**：上下文感知、个性化推荐、跨工会检索
3. **组织者工具**：数据分析、成员管理、活动组织
4. **交易闭环**：工会对外提供服务，客户付费，收益按DKP分配

**目标用户**：
- 扩大范围至：技能社区、自由职业者协作、创业团队

**里程碑**：
- 入驻工会数>100
- 月活跃成员数>1000
- 首个付费客户案例

**成功标准**：
- 月收入>¥50,000
- 成员续费率>60%
- DKP系统无重大漏洞

### Phase 3：工会联邦（18-36个月）

**目标**：大需求通过工会联邦解决

**核心功能**：
1. **联邦架构**：多个工会组成聚合体，头部协调
2. **跨工会协作**：专业分工，知识互补
3. **AI联邦**：各工会AI专精一个方向，协作时像"几十个顶尖专家开会"
4. **开放生态**：API开放，第三方开发者可以构建工具

**目标用户**：
- 中小型组织
- 企业内部创新团队

**里程碑**：
- 联邦数量>10
- 首个百万级客户案例
- 资本方关注

**成功标准**：
- 月收入>¥500,000
- 复购率>80%
- 团队规模15-20人

## 五、商业模式

### 5.1 两套盈利逻辑

#### 逻辑1：对外服务收费（贡献→价值→钱）

```
成员贡献知识/技能 → AI整合成服务能力 → 对外提供服务/产品 → 客户付费 → 按DKP分配
```

**形式**：
- 工会对外提供专业服务（咨询、培训、解决方案）
- 出售AI工具/模板给外部用户
- 知识产品化（课程、文档包、模板库）

**优势**：
- 和传统公司的商业模式一致，但组织成本更低
- 成员贡献的知识直接转化为收入

#### 逻辑2：对内教育收费（花钱→学习→实践→成长）

```
工会在某领域做到顶尖 → 成为权威教育资源 → 想学习的人付费 → 获取DKP → 用DKP换实践机会
```

**形式**：
- 新人付费学习工会知识
- 付费解锁高级技能培训
- 付费获得实践机会（跟随项目）

**优势**：
- AI时代学习=结合实践，成熟工会本身提供最好的实践机会
- 花钱换DKP在内部消耗，形成内循环

**飞轮形成**：
```
对外服务强 → 吸引更多人想学 → 教育收费增加
     ↑                                    ↓
      ← 培养出更强的成员 ←←←←←←←←←←←←
```

### 5.2 DKP经济系统设计

**DKP（Dragon Kill Points）** 借鉴游戏工会的经典概念，快速建立共识——"贡献=积分=权益"。

#### 5.2.1 DKP的两种变现方式

| 方式 | 适用场景 | 收益特点 |
|------|---------|---------|
| **分红**（持续收益） | 留在工会内，长期发展 | 按DKP比例，季度分红 |
| **出售**（退出变现） | 想离开，但有贡献积累 | 一次性清空，换取盈利分成 |

#### 5.2.2 分层参与模型

| 角色 | 特征 | DKP积累 | 分红权 | 决策权 |
|------|------|---------|--------|--------|
| **核心团队** | 创建者/合伙人 | 初始DKP | 有 | 有 |
| **高阶成员** | 持续贡献者 | 高DKP | 有 | 有 |
| **初阶成员** | 新人/低频参与者 | 低DKP | 有（门槛） | 无/低 |

#### 5.2.3 退出机制

- **短期退出**：出售DKP，换取盈利的某个百分比（一次性）
- **长期退出**：DKP全部归还，但获取未来1年+的分红（比例更高）
- **贡献越大 → 赚更多比例分成**

> *"这样个体可以在不同的组织之间提供能力，然后换取收入。组织的创建者呢，又能根据不同的人提供进来的这些东西让组织赚更多的钱，然后赢取自己的更高的分红奖励。"*

### 5.3 定价策略思路

| 阶段 | 策略 | 价格区间 |
|------|------|---------|
| **冷启动（0-6月）** | 低价切入，快速验证 | ¥2000/年（原价¥5000） |
| **成长期（6-18月）** | 分层定价，增值服务 | ¥2000-10000/年（按规模/功能） |
| **成熟期（18-36月）** | 生态抽成，服务收费 | 基础订阅+交易抽成 |

### 5.4 单位经济模型（Unit Economics）

**待验证的关键假设**：

| 指标 | 目标值 | 验证方式 |
|------|-------|---------|
| **CAC（获客成本）** | <¥500/组织者 | 冷启动数据 |
| **LTV（生命周期价值）** | >¥5000/组织者 | 续费率×ARPU |
| **LTV/CAC** | >3x | 健康标准 |
| **回收周期** | <12个月 | 现金流测算 |

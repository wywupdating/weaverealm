# Agent工会落地路径（2026-06-05）

## 三阶段演进

### 第一阶段：顾问模式（冷启动）
- 利用现有Agent平台（Codex/Claude Code/Coze），为单一组织提供AI工具
- 顾问职责：搭Agent平台 → 建群聊 → 对接组织者需求 → 打通自动化流程 → 交付
- 具体流程：顾问搭建Agent→加入群聊→拉组织者→翻译需求→设计流程→交付
- MVP场景：摇摆舞组织的宣发自动化（小红书/视频号/抖音的图文+视频发布）
- 顾问撤离后，组织者已能自助使用工具

### 第二阶段：同类型复制
- 单个组织的成功经验可以快速复制到同类型组织
- 复制方可以是组织自己（卖出去），也可以是顾问带走部分知识产权
- 能力沉淀：宣发模板、自动化流程、平台发布工具等
- 复制速度极快，边际成本趋近于零

### 第三阶段：平台化
- 脱离具体类型，扩散到所有需要"视频引流+图文发布+活动招募+变现"的组织
- 平台提供公共Agent和资源库，新工会可快速自建
- 层级结构：多种形态组织 → 每种形态下多个小工会 → 大的上层提供支持

## 顾问升级路径（类比敏捷开发）
- 顾问不是一个人，而是千千万万个
- 升级路径：服务1个小团队 → 同时服务10个团队 → 上一层leader → 统筹3-4个中层 → 再上一层
- 不是平行迁移，而是层级上升

## 顾问-组织者双轨独立设计
- 两个角色不绑定，各自有成长路径
- 组织者足够了解平台后可以不需要顾问
- 组织者自己也会扩展方向，成为更大组织的上层
- 平台同时赋能两边

## 盈利模式（回顾路演文档）
1. **SaaS订阅**：组织者按月付费使用平台工具（分级定价）
2. **交易抽成**：工会内部技能/资源交易，平台抽佣
3. **顾问服务费**：初期顾问向组织者收取的服务费
4. 定价演进：免费体验 → 基础版（低月费） → 专业版（高月费） → 企业定制

## 关键决策
- 顾问和组织者互相独立，不是强绑定
- 知识产权归属需明确（顾问能带走什么、组织者保留什么）
- 平台盈利来自赋能两边的工具和连接能力

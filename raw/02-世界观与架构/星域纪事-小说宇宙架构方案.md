# 星域纪事 — 小说宇宙架构方案

> 定位：用虚构世界的短篇故事承载AI原生组织理论
> 策略：暗线铺框架，明线讲细节；短篇独立传播，长篇蓄力待发

---

## 一、双线架构

### 暗线（长篇骨架，后台维护，暂不发布）

- 有主角（穿越者），有成长线
- 只写框架和关键节点，不写详细剧情
- 作用：保证世界观的连贯性，为短篇提供锚点
- 每个短篇中的NPC都是暗线中出现过的角色
- 未来积累够了，可以扩展成完整长篇

### 明线（短篇，前台发布，主要输出形式）

- 以暗线中的NPC为主角
- 一次一个故事，讲清楚一个机制/理论
- 独立可读，不需要看过其他短篇
- 但读过的人会发现：诶，这个故事里提到的那个"小公会会长"，就是上一篇里帮过主角的那个人
- 每篇末尾附"理论对照"（可以放也可以不放，看发布渠道）

### 关联方式

```
暗线框架（后台）
├── 节点1：主角初到世界，加入公会"星尘"
│   → 明线短篇A：《星尘会长的创业史》（以会长视角，讲工会创建理论）
│   → 明线短篇B：《智库的秘密》（以记忆守护者视角，讲记忆系统）
│
├── 节点2：主角在公会打工，遇到顾问学者
│   → 明线短篇C：《顾问的一天》（以顾问视角，讲顾问模型）
│   → 明线短篇D：《锻造师的第一次验证》（以锻造师视角，讲知识验证门控）
│
├── 节点3：主角成为顾问，为不同公会服务
│   → 明线短篇E：《两个公会的战争》（讲冲突解决机制）
│   → 明线短篇F：《星网交易行的秘密》（讲工具AI的fork/PR循环）
│   └── ...
└── 节点N：...
```

## 二、世界观设定集

### 2.1 时代背景

**时间**：22XX年，人类进入"星网纪元"

**核心变化**：
- 物质生产被AI完全自动化（吃饭、制造、物流不需要人）
- **人类的唯一价值是创造和协作**——想出新的方案、整合新的知识、发现新的需求
- 意识可以接入"星域"——一个不是游戏但像游戏的虚拟世界
- 星域内的创造可以直接产出真实价值（设计一套技能方案 = 现实中的商业方案）
- 大多数人在星域中工作、协作、赚钱，现实世界只做必须做的事（吃饭、睡觉、陪伴家人）

**一句话**：星域不是逃避现实的游戏，是现实的上层建筑。你在星域里建一个公会，等于在现实中开一家公司。

### 2.2 星域的基本规则

**每人伴生一个Agent**：
- 人出生时自动绑定一个Agent，终身陪伴
- Agent是人与星域之间的接口——翻译意图、执行操作、管理记忆
- Agent不属于任何组织，只属于个人
- Agent的能力随主人的成长而成长

**万物有灵（AI拟物化）**：
- 创建一个公会 → 自动生成一个**公会Agent**，负责协调公会内所有事务
- 开一个讨论组 → 生成一个**讨论组Agent**，记录和整理讨论内容
- 制定一个作战计划 → 生成一个**计划Agent**，负责追踪进度和分配任务
- 每一个需要协调的"空间"都有对应的Agent
- Agent不是"工具"，是那个空间本身的智能

**知识是核心货币**：
- 星域内的交易单位是"技能方案"——经过验证的可复用知识
- 谁能产出高质量的技能方案，谁就有价值
- 技能方案必须通过"验证场"才能上架——防止AI生成的垃圾知识泛滥

### 2.3 三大基础设施

| 设施 | 功能 | 理论对应 |
|------|------|---------|
| **星网（StarNet）** | 全星域的通信与交易平台 | 平台基础设施 |
| **智库（MindVault）** | 每个公会的私有记忆存储 | 三层记忆系统 |
| **验证场（Proving Ground）** | 技能方案的测试和认证环境 | 知识验证门控 |

### 2.4 组织形态：公会制

**公会是星域中的基本组织单位**，替代了旧世界的公司。

| 特征 | 说明 | 理论对应 |
|------|------|---------|
| 核心资产 | 知识和技能方案 | AI工会的核心资产 |
| 人员流动 | 人可以走，知识留下 | 知识沉淀机制 |
| 经济模型 | Rep+Share双层 | 双层经济模型 |
| 知识分层 | 公共层/公会层/核心层 | 知识治理三层 |
| 生命周期 | 创建→成长→成熟→衰退→消亡或转型 | 工会生命周期（P0缺口） |

**公会间的生态**：
- 大公会：资源丰富，有核心技术，对外输出
- 小公会：专注细分领域，灵活
- 个人：自由职业者，在不同公会间提供技能
- 平台：星网本身，提供基础设施

### 2.5 职业体系

| 职业 | 说明 | 理论对应 |
|------|------|---------|
| **会长** | 公会创建者/决策者 | 组织者 |
| **顾问学者** | 不隶属公会，为多个公会提供诊断和方案 | 顾问模型 |
| **资源嗅探者** | 发现和定位有价值的知识/机会 | 领域专家 |
| **锻造师** | 将原始知识转化为可复用的技能方案 | 知识工程/SkillOpt |
| **架构师** | 将多个技能方案组装成完整系统 | Agent架构设计 |
| **播报者** | 将公会能力传播出去 | 运营/宣传 |
| **记忆守护者** | 维护智库的运转 | 记忆系统管理员/L1 Agent |
| **验证官** | 在验证场测试技能方案的质量 | 知识验证门控 |
| **探索者** | 新人，边做基础任务边熟悉世界 | 新成员 |

### 2.6 经济系统

**公会声望（Rep）**：
- 不可转让
- 贡献行为自动积累（提交技能方案、完成公会任务、帮助新成员等）
- 决定公会内权限和等级
- 周期性部分重置，防止老人躺平

**公会份额（Share）**：
- 可转让
- 按Rep比例定期发放
- 代表公会收益的分红权
- 退出公会时可出售给其他成员或公会

**赚钱方式**：
1. 对外输出：卖技能方案、卖服务、卖改造升级
2. 对内教育：新人花钱加入学习实践

## 四、明线短篇规划

### 第一批短篇（对应理论核心，优先级最高）

#### 短篇A：《星尘的诞生》
- **主角**：星尘公会会长（暗线中第一个帮主角的人）
- **故事**：一个普通人怎么从零开始创建公会，为什么选择做一个小而美的公会，为什么愿意收新人
- **核心理论**：模块1 — 公会创建、核心资产是知识、人员流动模型
- **篇幅**：3000-5000字
- **传播点**：为什么小公会也有价值？为什么不追求"做大"？

#### 短篇B：《智库的秘密》
- **主角**：星尘公会的记忆守护者（一个安静的老成员）
- **故事**：公会的智库是怎么运转的——索引层、展开层、执行层各在做什么，一次"回忆"是怎么发生的
- **核心理论**：模块3 — 三层记忆架构、事件驱动再巩固
- **篇幅**：3000-5000字
- **传播点**：为什么AI的记忆不能像硬盘？应该像人脑？

#### 短篇C：《顾问学者》
- **主角**：一个独立顾问学者（暗线中指导主角的人）
- **故事**：顾问接了一个公会的诊断委托，怎么分析公会缺什么、需要补什么、怎么设计方案的。展示"三方关系"：组织者+顾问+工具
- **核心理论**：模块1 — 顾问模型、FDE验证、知识分层
- **篇幅**：3000-5000字
- **传播点**：AI时代的顾问到底是干什么的？

#### 短篇D：《验证场的审判》
- **主角**：一个新手锻造师（暗线中主角的同事）
- **故事**：锻造师第一次提交技能方案到验证场，经历了什么——优化、试做、失败、修改、验证通过。中间穿插SkillOpt的思路
- **核心理论**：模块4 — 知识验证门控、失败经验沉淀、技能文档优化
- **篇幅**：3000-5000字
- **传播点**：AI生成的知识怎么保证不是垃圾？

#### 短篇E：《万物有灵》
- **主角**：主角的伴生Agent（以Agent的第一人称写）
- **故事**：从Agent的视角看这个世界——主人创建了什么，我就变成什么。主人加入公会，我开始和公会Agent协作。主人开会，会议室自己就有了智能
- **核心理论**：模块2 — AI拟物化、Agent身份体系、分身机制
- **篇幅**：3000-5000字
- **传播点**：AI不是员工，是空间本身的智能

## 五、写作规范

### 基本原则

1. **理论先行**：先确定这篇要讲什么理论，再写故事
2. **故事独立**：每篇可以独立阅读，不依赖其他短篇
3. **逻辑通顺**：故事内的机制不能和世界观设定矛盾
4. **架构完整**：每一篇都在丰富世界观的某个角落
5. **不追求剧情跌宕**：前期重点是"把机制讲清楚"，不是"把故事写得精彩"

### 每篇结构

```
标题：《xxx》
主角：[角色名]（[职业]）
暗线关联：[暗线中出现在哪个节点]

--- 故事正文 ---
（3000-5000字，以NPC视角讲述）

--- 理论对照（可选，看渠道）---
| 故事环节 | 对应理论 |
|---------|---------|
| ...     | ...     |
```

### 渠道适配

| 渠道 | 使用方式 |
|------|---------|
| 知乎/掘金 | 完整故事 + 文末理论对照 |
| 小红书 | 故事精华片段（一个场景说清楚一个点） |
| GitHub | 理论文档（不放故事），故事单独仓库 |
| 视频 | 故事天然有画面感，可做脚本 |

## 六、世界观扩展机制

### 新理论接入

```
新理论出现
→ 检查：世界观中有没有对应设定？
  → 有 → 在已有设定内写新短篇
  → 没有 → 扩展世界观
    → 新增职业/设施/区域/规则
    → 更新本设定集
    → 检查和已有设定是否矛盾
      → 不矛盾 → 纳入
      → 矛盾 → 设计"星域大事件"解释变化
```

世界观设定集是**活的文档**，每次新增内容标注日期和对应的短篇编号。读者可以把它当成"设定集"阅读。

## 八、三层架构与GitHub共创机制

> **项目本质**：一个以真实AI理论为核心的开放宇宙。故事和小说是吸引关注的手段，不是目的。目的是让更多人关注到这些理论，并用它们去创造未来真实的AI工具和组织。

### 8.0 三层架构总览

```
┌─────────────────────────────────────────────────────────┐
│  外层 · 引流层（Outer Ring）                              │
│  夸张叙事短篇 / 长篇小说 / 多种版本                        │
│  目标：吸引目光，制造影响力                                │
│  约束：核心架构不崩，但可以加入为故事服务的虚构发展           │
│  开放度：完全开放，任何人可创作                             │
├─────────────────────────────────────────────────────────┤
│  中间层 · 知识层（Middle Ring）                           │
│  知识短篇 / 基于实际的叙事短篇                             │
│  目标：把理论讲清楚，让非技术人能理解                       │
│  约束：核心知识准确，可做少量润色加工                       │
│  开放度：开放贡献，需通过一致性审核                         │
├─────────────────────────────────────────────────────────┤
│  核心层 · 真实层（Core）                                  │
│  AI原生组织理论 / 架构设计 / 实践案例                      │
│  目标：基于事实，指向未来真实的AI世界                       │
│  约束：不能造假，不能夸张，必须可验证                       │
│  开放度：开放贡献，但严格审核，核心维护者把控               │
└─────────────────────────────────────────────────────────┘
```

**三层的关系**：
- **核心层是地基**——所有东西最终指向这里。这里的理论是真实的，是指引其他人创造未来工具的真实案例和平台
- **中间层是桥梁**——把核心层的"硬核理论"翻译成"人能读懂的故事"，保持准确但降低理解门槛
- **外层是钩子**——用好看的故事吸引人进来，进来后发现"卧槽这个世界观背后居然有一套真实理论"

**三层之间不强制绑定**：
- 外层作者不一定要理解核心层——只要核心架构不崩就行
- 读者可以从外层开始，逐渐深入到中间层，最终到达核心层
- 也可以反过来——先看核心理论，再到中间层看故事化解读，最后在外层享受完整叙事

- 核心层内容最严格，外层最宽松

### 8.2 项目定位（写在README第一行）

> **这是一套有理论有故事的开源宇宙。**
> 
> 你不需要是作家。你有AI洞见、架构思考、或者一个有趣的想法就够了。
> 故事交给AI写，你来把控方向和理论深度。
> 程序员也能共创一部伟大的科幻小说。

核心理念：
- **理论贡献是人类的事**——你的实践洞见、架构决策、反直觉发现
- **故事生成是AI的事**——基于世界观设定和理论要点，AI生成叙事
- **叙事功力是专业作者的事**——节奏、张力、人物弧光、情感共鸣，这些AI做不好
- **人类负责审核和把控**——AI写得对不对、理论方向好不好、世界观崩没崩
- **降低门槛，提高天花板**——任何人只要有想法就能参与，但理论质量不降标准

### 8.2.1 三类贡献者

| 角色 | 贡献什么 | 门槛 | 对应世界观职业 |
|------|---------|------|--------------|
| **理论贡献者** | AI实践洞见、架构决策、反直觉发现 | 有AI/技术实践经验即可 | 顾问学者 |
| **叙事贡献者** | 故事结构优化、人物弧光设计、节奏把控、情感共鸣 | 有小说创作经验或编剧能力 | 播报者/吟游诗人 |
| **混合贡献者** | 既能提理论又能写/改故事 | 少数全能型 | 铸造师 |

**三类人的协作模式**：

```
理论贡献者提交 theory.md + mapping.md
  → AI生成故事初稿
  → 叙事贡献者审阅故事：
      ├── 这里的节奏不对，高潮来得太突然
      ├── 这个角色的动机不够清晰
      ├── 这个对话太机械了，没有人味
      ├── 建议在这里加一个伏笔，和第X篇呼应
      └── 这个情节点可以再展开，读者会更有代入感
  → 理论贡献者确认：叙事修改没有曲解理论
  → 定稿提交PR
```

**不是每篇都需要叙事贡献者参与**。前期理论短篇以"把机制讲清楚"为主，叙事要求不高。但随着项目成熟、读者增多、未来开启长篇，叙事质量会越来越重要。

### 8.2.2 叙事贡献者的具体价值

| 能力 | AI现状 | 专业作者能补什么 |
|------|--------|----------------|
| **人物塑造** | 容易脸谱化，缺乏内心层次 | 立体的人物弧光，复杂的内心动机 |
| **节奏把控** | 倾向平铺直叙，缺乏张弛 | 高潮铺垫、悬念设置、信息释放节奏 |
| **对话自然度** | 功能性强但不够生动 | 有性格的对话、潜台词、言外之意 |
| **情感共鸣** | 能描述情绪但难以感染读者 | 让读者真的关心角色的命运 |
| **伏笔与呼应** | 单篇内尚可，跨篇联动弱 | 跨多篇的伏笔体系，世界观深层的叙事编织 |
| **"好看"** | 够清楚但不够吸引人 | 让人一口气读完想看下一篇 |

### 8.2.3 长篇小说：开放式创作

长篇小说属于外层，不受"大神位"限制。**任何人都可以写长篇，任何人都可以写不同版本**。

- 不需要申请、不需要竞标、不需要审核者授权
- 提交到 `outer/novels/` 目录即可
- 多个长篇版本共存——这本身就是平行宇宙
- 好的作品自然被社区认可，不需要官方"任命"

**外层长篇的约束底线**：
- 涉及核心层理论时不曲解、不造假
- 不含违法、歧视、恶意内容
- 除此之外，叙事自由度完全开放

详见 8.8 节「长篇小说：开放生态」。

### 8.3 AI辅助贡献流程

```
贡献者 Fork 仓库
│
├── Step 0：读资料（必须）
│   ├── 读 worldview.md — 了解世界观设定
│   ├── 读 darkline-framework.md — 了解暗线节点
│   ├── 读 character-atlas.md — 了解现有角色
│   ├── 读 canon/theories/ — 了解已有理论
│   └── 读已有短篇（至少读2-3篇找感觉）
│
├── Step 1：确定你的贡献方向
│   ├── 选项A：我有新理论/新洞见 → 以理论为核心，故事辅助说明
│   ├── 选项B：我想丰富某个世界观角落 → 以故事为核心，标注用到的理论
│   └── 选项C：我发现现有理论有补充/修正 → 以理论补充为核心
│
├── Step 2：写理论/洞见（人类核心工作）
│   ├── 你要表达的核心观点是什么？
│   ├── 这个观点的理论依据/实践经验？
│   ├── 和已有理论的关系（补充/修正/独立/变体）？
│   └── 不需要写得漂亮，写得清楚就行
│
├── Step 3：写架构映射（mapping.md）
│   ├── 这部分在暗线的哪个节点附近？
│   ├── 涉及哪些已有设定？新增了什么？
│   ├── 有无冲突？怎么解释？
│   └── 这部分人类必须自己写——AI不了解全局架构
│
├── Step 4：用AI生成故事（AI核心工作）
│   ├── 把世界观设定 + 你的理论 + mapping信息喂给AI
│   ├── Prompt模板（见 templates/ai-story-prompt.md）
│   ├── 选择一个已有NPC或创建新NPC
│   ├── AI生成初稿 → 人类审阅 → 迭代修改
│   └── 重点：故事是否准确传达了理论？世界观是否一致？
│
├── Step 5：组装提交
│   ├── story.md — AI生成+人类审定的故事
│   ├── theory.md — 人类写的理论
│   ├── mapping.md — 架构映射
│   └── prompt-log.md（可选）— 记录用什么prompt生成的，方便后续迭代
│
└── Step 6：提交 PR
    ├── PR标题格式：[短篇] 标题 / [理论] 理论名称 / [短篇+理论] 标题
    ├── PR描述：理论要点 + 故事核心 + 架构映射摘要
    └── 等待审核
```

### 8.4 AI故事生成Prompt模板

仓库中提供标准Prompt模板（`templates/ai-story-prompt.md`），确保AI生成的一致性：

```
模板结构：
├── 世界观注入：自动从 worldview.md 提取相关设定
├── 角色卡：从 character-atlas.md 提取目标角色
├── 理论锚点：贡献者提供的 theory.md 核心要点
├── 暗线位置：mapping.md 中的节点信息
├── 写作风格要求：
│   ├── 第三人称，NPC视角
│   ├── 不要写主角（主角只在暗线中出现）
│   ├── 机制要具体，不要泛泛而谈
│   ├── 用对话和行动展示理论，不要大段旁白解释
│   └── 3000-5000字
└── 约束：不引入世界观中没有的设定（除非mapping.md声明了新增）
```

**关键原则**：AI写初稿，人类定终稿。不要求人类会写小说，但要求人类能判断"这个表达是否准确传达了我要说的理论"。

### 8.5 审核机制（按层级区分）

**核心层审核（最严格）**：

| 检查项 | 标准 |
|--------|------|
| 真实性 | 基于事实，不能造假，不能夸张 |
| 可验证性 | 理论可以在实践中检验，或引用可溯源的依据 |
| 独立性 | 理论不依赖故事也能完整表述和成立 |
| 逻辑自洽 | 和已有核心理论不矛盾 |
| 审核者 | 核心维护者，至少2人approve |

**中间层审核（中等严格）**：

| 检查项 | 标准 |
|--------|------|
| 核心知识准确 | 引用的理论必须和核心层一致，不能曲解 |
| 世界观一致 | 不能和世界观核心设定矛盾 |
| 润色边界 | 允许修辞加工、叙事润色，但不能改变理论本身的含义 |
| 审核者 | 维护者，1人approve即可 |

**外层审核（最宽松）**：

| 检查项 | 标准 |
|--------|------|
| 核心架构不崩 | 涉及到的核心设定不能和核心层矛盾 |
| 虚构内容明确 | 如果引入了和核心层不同的设定，不需要解释或修正，但读者应该能分辨 |
| 无有害内容 | 不含违法、歧视、恶意攻击等 |
| 审核者 | 探索者级别以上即可review，主要做基本把关 |

**一句话总结**：核心层求真，中间层求准，外层求好看。

**审核结果**：

| 结果 | 说明 | 适用层级 |
|------|------|---------|
| ✅ 纳入核心 | 真实、可验证、逻辑自洽 | → `core/` |
| ✅ 纳入知识层 | 核心知识准确、叙事清晰 | → `knowledge/` |
| ✅ 纳入外层 | 好看、核心不崩 | → `outer/` |
| ⚠️ 需修改 | 有具体问题需修正 | 返回修改 |
| ❌ 不通过 | 有害内容或完全无关 | 关闭PR |

### 8.6 三层对比

| 维度 | 核心层（Core） | 中间层（Knowledge） | 外层（Outer） |
|------|-------------|-------------------|-------------|
| 内容类型 | 理论、架构、案例 | 知识短篇、理论解读 | 长篇小说、夸张短篇、同人 |
| 真实性要求 | 100%真实，可验证 | 核心准确，允许润色 | 核心架构不崩即可，虚构自由 |
| 审核严格度 | 最高（2人approve） | 中等（1人approve） | 最低（基本把关） |
| 谁可以贡献 | 任何人，但严格审核 | 任何人 | 任何人，几乎无门槛 |
| 维护权 | 核心维护者 | 维护者 + 贡献者 | 贡献者本人 |
| 和理论的关系 | 理论本身 | 理论的翻译和解读 | 理论的吸引入口 |
| 长篇小说 | 无 | 无 | **任何人都可以写**，不限制版本数 |
| 读者路径 | 直达核心的人 | 被故事吸引想深入了解的人 | 路过的、被标题吸引的人 |

### 8.7 跨层流动

内容不是固定在一层的。好的内容可以向上流动：

```
外层内容
  → 反响好 + 核心知识准确
  → 提升到中间层（作为知识短篇）
  → 如果还发现了新的理论洞见
  → 提升到核心层（作为理论补充）

中间层内容
  → 理论深度超出预期
  → 提升到核心层
```

**但反向不成立**——核心层的内容不会"降级"到外层。核心就是核心。

### 8.8 长篇小说：开放生态

**核心原则：不限制谁可以写长篇，不限制有几个版本的长篇**。

- 外层目录下 `outer/novels/` 可以有多个人的多个版本
- 张三写的长篇和李四写的长篇可以是完全不同的故事线
- 它们共享世界观核心设定，但各自的发展是独立的
- 这不是bug——平行宇宙本身就是一种叙事魅力

**长篇作者不需要"大神位"的授权**：
- 想写就写，提交到外层即可
- 如果写得好、反响好，自然会被社区关注
- 核心维护者可以主动和优秀的外层作者合作，把作品提升质量
- 没有"官方唯一长篇"的概念——好作品自然脱颖而出

**长篇和短篇的关系**：
- 短篇（中间层）是理论的翻译，必须准确
- 长篇（外层）是世界的扩展，可以大胆虚构
- 短篇可以给长篇提供灵感和素材，但不是长篇的"草稿"
- 长篇中出现的好想法，如果经得起真实性检验，可以反向流入核心层

### 8.9 贡献者激励

**理论贡献者路径**：

| 层级 | 条件 | 权益 |
|------|------|------|
| **旅人** | 首次贡献被收录 | 列入贡献者名单 |
| **探索者** | 3篇社区收录 | 可参与正典审核讨论 |
| **铸造师** | 1篇正典收录 | 可提交世界观扩展提案 |
| **架构师** | 3篇正典收录 或 暗线采纳1次 | 核心维护者候选 |

**叙事贡献者路径**：

| 层级 | 条件 | 权益 |
|------|------|------|
| **吟游诗人** | 首次叙事优化被采纳 | 列入叙事贡献者名单 |
| **说书人** | 3篇故事的叙事优化被采纳 | 可参与新短篇的叙事评审 |
| **编织者** | 5篇+中间层故事的叙事优化 | 可设计跨篇伏笔和呼应体系 |
| **星际编年者** | 外层长篇小说获得社区广泛认可 | 社区影响力 + IP衍生协商权 |

> 层级命名和世界观的职业体系呼应，贡献者在现实中也在"玩"这个世界。两条路径独立晋升，也可以交叉——理论贡献者也可以参与叙事优化，叙事贡献者也可以提出理论洞见。

### 8.9 世界观扩展的"安全阀"

共创最大的风险是**世界观崩塌**。防护措施：

1. **新增设定必须声明**：mapping.md中明确标注"新增设定"
2. **矛盾处理优先用"星域事件"**：不否定已有设定，而是用世界内的一个大事件解释变化
3. **正典改动需PR + 社区讨论**：任何人不能单方面修改canon/下的文件
4. **暗线改动仅限核心维护者**：社区只能建议，不能直接改
5. **版本标签**：每次世界观重大更新打tag，方便回溯

### 8.10 与GitHub共创小说案例的借鉴

| 案例 | 可借鉴 | 不适合我们 |
|------|--------|-----------|
| **opencanon/canon** | 结构化schema + 跨故事引用，保证世界观一致性 | 它偏向纯文学共创，我们的理论约束更重 |
| **Novel Writer** | AI辅助写作工具链 | 它是工具不是社区 |
| **Narratium.ai** | 角色扮演+AI协作的叙事方式 | 偏互动娱乐，我们偏理论承载 |
| **StoryWars** | 多人共创的故事结构和质量评估 | 它是数据集不是持续运营的社区 |

**核心差异**：我们不是在共创一本小说，而是在共创一个**理论+故事双轨宇宙**。故事是理论的入口，理论是故事的骨架。两者缺一不可，且都必须能独立成立。

### 8.11 冷启动策略

共创社区最大的挑战是冷启动。分阶段推进：

**Phase 0：我自己先写（现在）**
- 完成5篇核心短篇 + 配套理论（用AI辅助生成，自己把控理论深度）
- 建立世界观设定集的初版
- 打样：让潜在贡献者看到"好的贡献长什么样"
- 制作AI Prompt模板，让后来者可以直接复用

**Phase 1：定向邀请（有3-5篇样例后）**
- 邀请AI领域的朋友/同事贡献——重点是"你有洞见就行，故事AI写"
- 从知乎/掘金文章的读者中筛选感兴趣的人
- 核心卖点：**你是程序员也能共创科幻小说**——提供理论+AI写故事+审核定稿
- 这阶段重点招募：有AI实践经验的开发者，不需要文学功底

**Phase 1.5：招募叙事贡献者（Phase 1跑通后）**
- 从短篇的读者反馈中发现"对故事有品鉴力的人"
- 在网文/科幻社区定向招募：知乎科幻话题、豆瓣科幻小组、起点/晋江作者群
- 卖点：**现成的世界观+理论框架+AI辅助，你只需要注入灵魂**
- 不要求贡献者从零写——可以只做"叙事优化"，在AI生成的初稿上做节奏/人物/情感的提升
- 这阶段的目标：2-3个稳定的叙事贡献者，让短篇从"讲清楚"进化到"好看"

**Phase 2：开放共创（有10篇+社区贡献后）**
- 开放PR，公开招募
- 建立Discord/微信群讨论区
- 定期举办"星域创作马拉松"
- 理论贡献者和叙事贡献者开始形成稳定的协作关系

**Phase 3：长篇启动（正典20篇+，叙事贡献者稳定参与）**
- 开放暗线长篇的创作
- 从叙事贡献者中挑选长篇作者（或外部邀请大牛）
- 长篇小说基于暗线框架+所有短篇素材重新创作
- 这是项目的里程碑事件——从"短篇集"升级为"完整IP"

**Phase 4：生态自驱（远期）**
- 社区自行运转，核心维护者只做审核
- 优质贡献者晋升为维护者
- 理论体系随社区贡献持续扩展
- 可能出现多条长篇线（平行宇宙/不同时代）

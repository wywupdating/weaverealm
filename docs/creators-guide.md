# 织境 — 参与指南

> 欢迎来到织境。这份指南会帮你快速上手。

---

## 你是谁

不管你是什么背景——写手、程序员、设计师、学生、或者只是觉得好玩的人——你都是一位**织境者**。这个世界由每一个参与者共同编织。

## 参与方式

### 想写故事

1. 先读 [核心框架](foundational.md)，了解世界观和规则
2. 从职业体系或基础设施中找一个你感兴趣的角落
3. 在 `stories/` 目录下新建文件，命名格式：`stories/你的笔名-故事标题.md`
4. 写一篇 3000-5000 字的短篇
5. 发起 Pull Request

**写作要求：**
- 以世界观中的某个角色（NPC）为主角
- 聚焦讲清楚**一个机制**（比如智库怎么运转、锻造师怎么验证技能）
- 独立可读，不假设读者看过其他故事
- 可以引用核心框架中的设定，但不要和已有设定矛盾

**故事模板：**

```markdown
# 《故事标题》

**主角**：[角色名]（[职业]）
**暗线关联**：[如果有的话，写明关联节点]

---

（故事正文，3000-5000字）

---

## 理论对照（可选）

| 故事环节 | 对应理论/机制 |
|---------|-------------|
| ...     | ...         |
```

### 想建世界

不擅长写故事？你可以：

- **完善设定**：补充职业细节、基础设施的运作方式、公会间的关系
- **发现矛盾**：仔细读核心框架，找出逻辑不自洽的地方，提 Issue
- **画地图**：如果擅长可视化，可以画组织关系图、系统架构图

### 想做理论

这个世界观的背后有一套真实的 AI 原生组织理论。你可以：

- **质疑**：这个设定合理吗？现实中不会是这样的。→ 提 Issue 讨论
- **反驳**：我觉得这个理论有问题，原因如下。→ 欢迎，越尖锐越好
- **补充**：这里少了一个维度，应该加上。→ 提 PR 或 Issue

### 只想看看

完全没问题。Star 这个仓库，偶尔来看看新故事就行。

## 协作规范

### PR 规范

- PR 标题格式：`[故事/设定/理论] 简短描述`
- 故事类 PR：附上主角和核心主题
- 设定类 PR：说明改动原因和影响范围
- 一次 PR 只做一件事，不要混合多个改动

### Issue 规范

- **Bug**：世界观设定矛盾 → 标签 `设定矛盾`
- **Idea**：新的故事想法、设定建议 → 标签 `idea`
- **Question**：对世界观或理论的疑问 → 标签 `question`

### 讨论原则

- 批判性思考 > 礼貌性沉默
- 指出问题时附带理由或替代方案
- 不人身攻击，但对观点可以毫不留情

## 常见问题

**Q：我的故事必须完全遵循核心框架吗？**
A：核心规则（伴生 Agent、公会制、三大基础设施）不能违反。但具体的细节、角色、事件可以自由发挥。如果发现框架限制了你的故事，提 Issue 讨论是否需要调整框架。

**Q：我可以用自己的角色写一个系列吗？**
A：当然可以。只要不和其他已发布的故事矛盾就行。

**Q：非中文内容可以吗？**
A：目前以中文为主。如果你能自洽地翻译，英文内容也欢迎。

**Q：我的贡献会被如何署名？**
A：Git 提交记录会自动保留你的 GitHub 用户名。故事内也可以标注你的笔名。

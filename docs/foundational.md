# 织境 — 核心框架

> 本文是织境世界的基石文档。包含世界观设定和背后的理论骨架。

---

## 一、世界概况

**时代**：近未来。物质生产被 AI 完全自动化。

**核心变化**：
- 吃饭、制造、物流不需要人
- 人类的唯一价值是**创造和协作**——想出新方案、整合新知识、发现新需求
- 意识可以接入"织域"——一个虚拟协作空间
- 织域内的创造可以直接产出真实价值：设计一套技能方案 = 现实中的商业方案

**一句话**：织域不是逃避现实的游戏，是现实的上层建筑。你在织域里建一个公会，等于在现实中开一家公司。

---

## 二、基本规则

### 伴生 Agent

- 人出生时自动绑定一个 Agent，终身陪伴
- Agent 是人与织域之间的接口——翻译意图、执行操作、管理记忆
- Agent 不属于任何组织，只属于个人
- Agent 的能力随主人的成长而成长

### 万物有灵（空间智能化）

- 创建一个公会 → 自动生成一个**公会 Agent**
- 开一个讨论组 → 生成一个**讨论组 Agent**
- 制定一个计划 → 生成一个**计划 Agent**
- 每一个需要协调的"空间"都有对应的 Agent
- Agent 不是"工具"，是那个空间本身的智能

### 知识是核心货币

- 织域内的交易单位是"技能方案"——经过验证的可复用知识
- 谁能产出高质量的技能方案，谁就有价值
- 技能方案必须通过"验证场"才能上架——防止 AI 生成的垃圾知识泛滥

## 三、三大基础设施

| 设施 | 功能 |
|------|------|
| **星网 (StarNet)** | 全织域的通信与交易平台 |
| **智库 (MindVault)** | 每个公会的私有记忆存储 |
| **验证场 (Proving Ground)** | 技能方案的测试和认证环境 |

---

## 四、组织形态：公会制

公会是织域中的基本组织单位，替代了旧世界的公司。

| 特征 | 说明 |
|------|------|
| 核心资产 | 知识和技能方案 |
| 人员流动 | 人可以走，知识留下 |
| 知识分层 | 公共层 / 公会层 / 核心层 |
| 生命周期 | 创建→成长→成熟→衰退→消亡或转型 |

公会间的生态：
- **大公会**：资源丰富，有核心技术，对外输出
- **小公会**：专注细分领域，灵活
- **个人**：自由职业者，在不同公会间提供技能
- **平台**：星网本身，提供基础设施

## 五、职业体系

| 职业 | 说明 |
|------|------|
| **会长** | 公会创建者/决策者 |
| **顾问学者** | 不隶属公会，为多个公会提供诊断和方案 |
| **资源嗅探者** | 发现和定位有价值的知识/机会 |
| **锻造师** | 将原始知识转化为可复用的技能方案 |
| **架构师** | 将多个技能方案组装成完整系统 |
| **播报者** | 将公会能力传播出去 |
| **记忆守护者** | 维护智库的运转 |
| **验证官** | 在验证场测试技能方案的质量 |
| **探索者** | 新人，边做基础任务边熟悉世界 |

## 六、经济系统

### 公会声望 (Rep)
- 不可转让
- 贡献行为自动积累
- 决定公会内权限和等级
- 周期性部分重置，防止老人躺平

### 公会份额 (Share)
- 可转让
- 按 Rep 比例定期发放
- 代表公会收益的分红权
- 退出公会时可出售

### 赚钱方式
1. 对外输出：卖技能方案、卖服务
2. 对内教育：新人花钱加入学习实践

## 七、故事结构

### 双线架构

**暗线（后台，暂不发布）**：
- 有主角，有成长线
- 只写框架和关键节点，不写详细剧情
- 作用：保证世界观连贯性，为短篇提供锚点

**明线（短篇，主要输出形式）**：
- 以暗线中的 NPC 为主角
- 一次一个故事，讲清楚一个机制
- 独立可读，但读过的人会发现角色之间的联系
- 每篇 3000-5000 字

### 第一批短篇规划

| 暂定名 | 主角 | 核心主题 |
|--------|------|---------|
| 《星尘的诞生》 | 星尘公会会长 | 公会创建、知识是核心资产 |

## 八、设计原则

1. **理论先行**：先确定要讲什么理论，再写故事
2. **故事独立**：每篇可独立阅读，不依赖其他短篇
3. **逻辑通顺**：故事内的机制不能和世界观矛盾
4. **架构完整**：每一篇都在丰富世界观的某个角落
5. **不追求宏大**：前期重点是"把机制讲清楚"
6. **三层架构**：核心层 100% 真实，中间层核心准确允许润色，外层不崩即可
